Intro

This project focuses on creating a web application that empowers users to input specific time windows and geographical locations to obtain insights into average trip costs and strategic recommendations for drivers to maximize profitability. The motivation behind this project stems from the increasing demand for data-driven decision-making in the ride-sharing industry. As the volume of data generated by these platforms continues to grow, there is a compelling need for efficient tools that can handle large datasets and extract meaningful insights.

The overarching goal is to provide ride-sharing companies with a robust solution that not only processes data efficiently but also translates it into actionable recommendations for drivers. By creating a user-friendly web application, we aim to provide access to valuable insights, enabling both technical and non-technical users to make informed decisions.

Future Work

Collecting and combining real-time data.

The dataset we utilized for now was collected from the New York Government website, more specifically, the NYC Taxi & Limousine Commission page. However, these data are fixed and immutable. It means that sometimes when we want to give real-time estimations to the passengers based on past information, it may seem a bit unreasonable to come up with real-time estimations using only the past data without using real-time information as the labels. During the process, we did actually try to find some real-time data from NYC authorities. For example, we have looked up MTA (Metropolitan Transportation Authority) and Google Maps API to get real-time location and traffic information and OpenWeatherMap API or Dark Sky API to get real-time weather information. However, most of them are not free and may need certain accounts and obtain access tokens. In the future, we can collaborate and utilize these data to give our users more realistic estimated real-time information if we have the opportunity.

May include more information.

for data processing or machine learning

Right now, in our data, we only found some basic attributes for each pickup information such as pick-up and drop-off time and locations, total amount for each ride,­­­­­­ pickup hours, etc. When we did the data processing and idealizing of our machine learning process, sometimes we struggled because it was hard to find some reasonable method to do machine learning either in linear regression, clustering, etc. In the future, if there is more information regarding each data (such as the weather of that specific date, the actual traffic for that location on that date, etc.), we can insert more labels inside our data processing or machine learning method and give our user more realistic estimations based on more information in our data.

May include a passenger tab or more functions

so that passengers can also use our project to find the nearest, most popular pickup spots, with how much waiting time for each waiting spot.

Due to the lack of attributes in our current dataset, if we want to include some common selection attributes for a taxi pickup application, it will be quite hard since we currently don’t have such attributes. The same questions come if we want to extend our project to include passengers and estimate the nearest pickup locations for passengers. However, if in the future, some datasets with more data attributes are found, we may have a chance to include a passenger tab so that passengers can also use our project.
